标注文件格式

annotations.json 需要包含以下信息：

{
  "images": [
    {
      "file_name": "phone1.jpg",
      "annotations": {
        "x": 0.1,      # 左上角x坐标 (0-1)
        "y": 0.2,      # 左上角y坐标 (0-1)
        "width": 0.3,  # 宽度 (0-1)
        "height": 0.4  # 高度 (0-1)
      }
    },
    ...
  ]
}

使用说明

  1. 输入手机品牌和型号（例如：APPLE IPHONE15）
  2. 调整手机屏幕比例（默认9:19.5）
  3. 在手机模型上按住鼠标左键并拖动来标记NFC区域（将显示为红色框）
  4. 点击"生成JSON"按钮获取结果
  5. 提交数据审核：
    • 点击"提交PR"按钮
    • 在打开的GitHub页面中确认文件路径（data/品牌/型号.json）
    • 点击"Propose new file"提交审核
  6. 如需重新标记，点击"重置"按钮
  1. 选择检测模式：
    • 传统模式：基于图像处理算法检测
    • AI模式：使用训练好的模型检测（需要先上传模型文件）
  2. 选择包含手机图片的文件夹进行上传
    • 文件夹结构应为：品牌/型号/图片文件
    • 支持的图片格式：PNG、JPG、JPEG
    • 图片应清晰显示手机正面
  3. 等待检测完成，查看结果：
    • 下载JSON结果文件
    • 复制结果到剪贴板
    • 查看检测详情和错误信息
  1. 选择图片文件夹：
    • 点击"选择图片文件夹"按钮
    • 选择包含需要标注的手机图片的文件夹
    • 支持批量导入多张图片
  2. 标注操作：
    • 在图片上按住鼠标左键拖动，框选NFC区域
    • 绿色框表示当前选择的区域
    • 右侧实时显示标注信息
  3. 图片导航：
    • 使用"上一张"和"下一张"按钮切换图片
    • 顶部显示当前进度（例如：1/10）
  4. 标注管理：
    • 点击"重置框选"可以重新标注当前图片
    • 完成所有标注后点击"生成数据集"
    • 将自动打包图片和标注信息为ZIP文件
  1. 准备训练数据：
    • 使用标注工具生成的数据集
    • 确保数据集包含足够的样本（建议>100张）
    • 图片应该具有代表性和多样性
  2. 上传数据集：
    • 选择包含已标注图片的ZIP文件
    • 系统会自动验证数据集格式
    • 显示可用的训练样本数量
  3. 训练过程：
    • 点击"开始训练"启动训练
    • 实时显示训练进度和指标
    • 可以随时暂停或停止训练
  4. 模型导出：
    • 训练完成后可导出模型
    • 导出的模型可用于批量检测
    • 包含模型文件和配置信息
